## 1. Executive Summary — Systemic Patterns

These recur across 4+ families — fixing one root cause pays off everywhere.

1. **The coarse-pointer 44px floor covers only 5 selectors — the single most pervasive gap.** `globalLayer.ts:222-224` `@media (pointer: coarse)` inflates only `.btn, .in, .select-trigger, .btn--icon, .btn--circle`. Everything else stays sub-44px on touch: `.toggle` (14-22px), `.check/.radio` (16px), `.slider` (3px track), `.segctrl__btn` (~28px), `.menu__item/.navsub__item` (28px), `.tab`, `.pagination button`, `.list__row`, `.tbl tbody tr`, `.calendar__cell`, `.lightbox__btn`, `.att-chip__x` (18px), `.pwinput__eye` (32px), close buttons, chart marks. **Fix:** expand the block to a documented roster (+ hit-expanding pseudo-elements for sub-token visuals like the toggle knob and 3px slider track).

2. **The keyboard contract lives in hand-written JSX, never in the recipe — and the canonical surface is often the broken one.** `useDropdown` (AppHelpers.tsx:21-39) only does outside-click + Escape — no roving tabindex, no arrow-key nav, no typeahead, no focus-trap, no focus-return. Hits Dropdown, Context menu, Navmenu, SplitMenu. The *gallery* ("source-of-truth") often regressed below its app twin (gallery sortable table vs app SortHeader; app CmdPalette dead vs gallery full combobox). **Fix:** one roving-tabindex + focus-return behavior; bring every canonical surface to its best in-repo twin; document the ARIA *in the recipe comment* so CDN consumers rebuild it.

3. **Focus is never returned to the trigger on dismiss.** No `.focus()` in `useDropdown`; Dialog, Sheet, Lightbox, Command palette, date-picker popover all drop focus to `<body>` on close. Highest-value a11y fix. **Fix:** capture trigger ref, restore focus in every close path.

4. **No press/`:active` tier across whole families.** `.btn/.slider/.kanban/.numinput` have `:active`; the entire nav family (`.menu__item, .navrow, .tab, .pagination button, .navmenu__item, .sidenav__toggle`), `.alert__close`, table rows, chart marks, `.dropzone` have none. The kit emits **no `--k-state-active`/pressed token at all**. **Fix:** mint `--k-state-press` (stronger color-mix than `--k-state-hover`) + apply `:active` family-wide.

5. **Status-by-color-alone, plus ARIA that lives only in demo JSX.** WCAG 1.4.1 misses: table sort (hue+rotation), usage-meter bands, timeline dots, chart series (color-only, worsened by user-chosen palette), `.meta-status` dots, stat-tile delta. `role`/`aria-live`/`aria-valuenow` for Alert, Banner, Progress, usage-meter, spinner exist only in preview JSX, not the recipe — a CDN consumer gets nothing. **Fix:** add a non-color channel (glyph/dash/label/weight); document live-region/value ARIA in each recipe header. Toast already does `role={tone==='error'?'alert':'status'}` (ComponentGallery:2175) — mirror onto Alert + Banner.

6. **Overlays animate IN but never OUT, and don't collision-handle.** `k-scale-out`/`k-fade-out` keyframes exist in globalLayer but are **never wired**; `--k-ease-in` has zero usages. Dialog, Sheet, Command palette, Popover unmount instantly. Every overlay uses hard-coded `left:0/top:calc(100%+8px)` with no flip/shift — Popover, Tooltip, context menu, navmenu, chart tooltips (comment literally claims "clamped inside" but isn't), date-picker all overflow at viewport edges. **Fix:** wire paired exit keyframes gated on a closing state; add edge-collision clamp (or Floating-UI-style flip/shift with `data-side`).

7. **iOS-zoom + field-family inconsistency in text inputs.** Every field uses `--k-type-small` (11.5-13px) < 16px → iOS Safari zooms on focus and never zooms back. Plus: native `select.select:focus` uses an **undefined** `--k-focus-ring` (falls to 22% halo) while siblings use `--k-ring-halo` (28%); wrappers (`.numinput/.pwinput/…`) skip the `:disabled` treatment plain `.in` gets, despite a comment claiming coverage. **Fix:** `@media (pointer:coarse)` `max(--k-type-small,16px)` on all fields; unify halo to `--k-ring-halo`; shared `:has(:disabled)`/`[aria-disabled]` wrapper rule.

8. **Duplicated motion keyframes + focus-ring color floor gap.** Three near-identical pulse keyframes (`meta-status-pulse`, `k-pulse`, a preview copy) + two shimmer (`sk` vs `k-shimmer`) drift independently. `--k-ring` is the **raw** primary (buildTokens.ts:641) with no 3:1 floor — a pale/low-chroma primary yields a sub-3:1 focus ring (geometry 2px+2px offset is fine). **Fix:** consolidate onto global keyframe tokens; clamp `--k-ring` to a contrast-checked ramp step ≥3:1.

## 2. Composition Verdict — Add / Remove / Merge

**ADD (missing primitives):**
- **Menubar** — no app-menubar primitive (the dropdown roving model is the substrate).
- **Resizable / split-pane** — absent; common in the dashboard/cockpit use-case this kit targets.
- **Standalone Collapsible** — Accordion is native `<details>`; no single-disclosure primitive.
- **RTL / Direction support** — no logical-property/`dir` story (att-chip recipe even *claims* a `direction:rtl` truncation it never implements).
- **Interactive/clickable Card** (`.card--interactive`) — exported `.card` ships no cursor/`:active`/focus; the only hover is preview-chrome (never exported).
- **Indeterminate Progress** (`.progress--indeterminate`) — determinate-only today.
- **Image-bearing Avatar** with `onError`→initials + `delayMs` — recipe is initials-only.
- **Dismissible Badge** variant (trailing aria-labeled ×).
- **Spinner size tiers** (sm/md/lg) — single fixed ~20px.

**MERGE / DEDUPE:**
- One **menu keyboard behavior** (roving tabindex + focus-return) shared by Dropdown, Context, Navmenu, SplitMenu.
- One **shimmer keyframe** (`k-shimmer`) + one **pulse keyframe** (`k-pulse`) — kill duplicates.
- One **focus-halo token** (`--k-ring-halo`) across `.in`, wrappers, `select.select`, `.otp__slot` — remove undefined `--k-focus-ring`.
- **Reconcile gallery ↔ app twins** (table sort header, command palette, empty/activity dot) onto the stronger implementation.

**REMOVE / FIX-OR-DELETE:**
- Dead `.accordion__head:focus-visible` rule (globalLayer:158) — DOM renders bare `<summary>`; retarget `.accordion summary:focus-visible`.
- Misleading bare `aria-selected` on `<tr>` in a non-`grid` table — drop or promote to `role="grid"`.
- Inaccurate recipe comments shipping as docs: button heights (`28/34/44` vs real `32/36/40`), att-chip `direction:rtl`, chart-tip "clamped inside."

**KEEP (genuine leads over the references — don't regress):** parametric `--k-pad`/`--k-space` card split; em-based `.badge--count` circle math; checkbox absolute-`::after` anti-jump tick; button-group `z-index`-on-focus; reduced-motion guard + HCM transparent-outline fallback + borders-off validation floor; coarse-pointer 44px (where it applies) leads shadcn; CSS-only `.barchart` is keyboard+SR-labelled by default (better than shadcn's opt-in).

## 3. Prioritized Fix-List (top 15, impact ÷ effort)

1. **Expand the coarse-pointer 44px floor** (`globalLayer.ts:222-224`) to the full interactive roster. *One block fixes ~20 findings across 6 families.*
2. **Add focus-return to `useDropdown`** (AppHelpers.tsx:21-39): capture trigger ref, `.focus()` on close. *Family-wide focus-loss fix.*
3. **Wire roving-tabindex + arrow-key nav into `.menu`** (focus first item, Arrow/Home/End, typeahead, trap). *Largest single a11y gap; reused by 4 surfaces.*
4. **`@media (pointer:coarse) font-size: max(--k-type-small, 16px)`** on all text fields. *Kills iOS focus-zoom.*
5. **Tone-driven `role`/`aria-live` on Alert + Banner** — mirror Toast (ComponentGallery:2175). *Trivial.*
6. **Floor `--k-ring`** (buildTokens.ts:641) to ≥3:1 vs `--k-surface/--k-bg`. *Fixes focus-ring contrast everywhere at once.*
7. **Wire paired exit keyframes** (`k-scale-out`/`k-fade-out`) for Dialog/Sheet/Popover/Command palette; activates unused `--k-ease-in`.
8. **Port app `SortHeader` into the gallery table** (`role=columnheader`, tabIndex, `aria-sort`, Enter/Space) — makes the dead `:focus-visible` rule live.
9. **Wire the CodeBlock copy button** (ComponentGallery:2912) — currently 100% non-functional (no `onClick`); add `--copied` swap + `aria-live`.
10. **Focus-trap + Escape + focus-restore + scroll-lock** on Dialog, Sheet, Lightbox (all assert `aria-modal=true` without the contract).
11. **`role=progressbar`+aria-value* on `.usage__bar`** (ComponentGallery:99) — copy the wired `.progress` pattern.
12. **Pause Toast auto-dismiss on hover/focus** (ComponentGallery:2151).
13. **Mint `--k-state-press` + apply `:active`** to the nav family + `.alert__close`.
14. **Unify field focus halo to `--k-ring-halo`** on `select.select` + shared wrapper `[aria-disabled]`/`:has(:disabled)` rule.
15. **Line-style variation (solid/dashed/dotted) for chart series** + propagate the `.barchart` keyboard pattern to ChartFrame marks + donut slices. *Highest-leverage colorblind/a11y win; palette is user-chosen so non-color channel is the only robust guard.*

*Near-zero-effort batch: dedupe shimmer (`sk`→`k-shimmer`) + pulse (`meta-status-pulse`→`k-pulse`) keyframes; fix stale recipe comments; fix dead `.accordion__head:focus-visible` → `.accordion summary:focus-visible`.*
